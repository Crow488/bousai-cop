## 成果物1: 避難所GeoJSON — data/shelters.geojson

- 対象自治体: 長崎県佐世保市（市区町村コード 42202）と 長崎県北松浦郡佐々町（42391）
- ソース: 国土地理院「指定緊急避難場所データ」の公開CSV（https://www.gsi.go.jp/bousaichiri/hinanbasho.html から辿れる公開データ）を第一候補にする。取得できなければ国土数値情報（P20 避難施設）等、官公庁の一次データのみ使用。個人サイトのコピーは使わない。
- 形式: GeoJSON FeatureCollection、座標系はWGS84（経度・緯度）。Point。
- 各featureのproperties:
  - name: 施設名
  - address: 所在地
  - muni: "佐世保市" または "佐々町"
  - hazards: 対応災害種別の配列（例 ["洪水","土砂災害","地震"]。元CSVの災害種別フラグ列から変換）
  - kind: "指定緊急避難場所"（指定避難所データも取れたら統合し kind を分ける。無理なら緊急避難場所のみで良い）
- 2自治体を1ファイルに統合。

## 成果物2: 行政界GeoJSON — data/boundaries.geojson

- 佐世保市と佐々町の行政界ポリゴン（属性 name に自治体名）。
- ソース候補: GitHub の smartnews-smri/japan-topography（国土数値情報N03由来）や niiyz/JapanCityGeoJson など、N03由来の信頼できるリポジトリ。もしくは国土数値情報N03を直接。
- 適度に簡略化してファイルサイズ1MB以下にすること（頂点間引き可。見た目の輪郭が保てれば良い）。

## 成果物3: 出典記録 — data/SOURCES.md

- 各ファイルについて: 取得URL、取得日時、元データの名称・提供元、ライセンス/利用規約の要点（出典明記義務など）を記録。

## 検証（必ずやる）

- 各GeoJSONがpythonのjson.loadでパースできること
- 件数を数え、全ポイントが経度128.5〜130.5 / 緯度32.5〜34.0 の範囲内にあること（範囲外があれば除外して報告）
- 最後に「件数・出典・保存先」を要約して報告すること
